FE가 하는 일

사용자의 행동을 받아 이벤트 처리, API 요청을 보내고, 응답에 맞춰 화면 상태 갱신 즉, FE는 사용자 경험이 끊기지 않도록 흐름을 연결하는 역할을 한다.

BE가 하는 일

DB 반영, 권한 확인, 실제 데이터 저장 등은 BE의 역할.

2. BE와의 통신은 어떻게?

FE와 BE는 API로 대화한다

BE와 협업을 할때 API 명세를 정의한다.

API란?

Application Programming Interface

쉽게 말하면, 프로그램끼리 대화할 때 사용하는
약속된 창구, 규칙, 요청 방식

왜 필요한가?

FE는 DB를 직접 건드리지 않는다.
필요한 데이터가 있으면 API 주소로 요청을 보내고, BE는 정해진 형식으로 응답을 돌려준다.

예시 API로 알아보자

아래는 JSONPlaceholder라는 공개 테스트 REST API를 사용. 키 없이 호출할 수 있고, GET/POST 예시를 보여주기 좋다.

직접 요청을 보내보자

버튼을 누르면 브라우저가 공개 API로 실제 요청을 보냄. 아래 요청과 응답이 즉시 바뀐다.

1. 프론트엔드가 요청 준비
method, URL, body, headers를 구성
2. 네트워크를 통해 전송
브라우저가 서버로 HTTP 요청을 보냄
3. API 서버가 응답
JSON 형태로 데이터를 반환

핵심

“FE는 API에 요청을 보내고, 응답으로 받은 JSON을 해석해서 화면에 반영.
즉 데이터를 직접 저장소에서 꺼내는 게 아니라, API를 통해 간접적으로 받는 구조”

아직 요청 전
대기 중

요청(Request)

WAIT 
버튼을 누르면
요청 정보가 여기에 표시됩니다.

응답(Response)

WAIT 
아직 응답이 없습니다.
fetch 예시 브라우저 코드 
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then((response) => response.json())
  .then((data) => console.log("fetch 결과:", data));

axios.get("https://jsonplaceholder.typicode.com/posts/1")
  .then((response) => console.log("axios 결과:", response.data));

요약

  • • GET은 데이터를 받는 예시
  • • POST는 데이터를 보내는 예시
  • • 응답은 보통 JSON으로 온다
  • • 프론트엔드는 이 JSON을 읽어 화면을 바꾼다

참고: 이 POST는 테스트 API이므로 실제 서버 DB에 영구 저장되지 않는다.

한줄 요약

API는 “FE가 BE 기능을 사용하기 위한 약속된 문”. FE는 API에 요청을 보내고, BE는 처리 결과를 JSON 등으로 응답.

3. 이전의 내용을 합친 웹 동작원리

URL 입력부터 화면이 보이기까지의 웹 동작원리

NAVER에 접속해볼까? www.naver.com을 입력했을 때 발생하는 일

1

URL 입력

사용자가 브라우저 주소창에 사이트 주소를 입력한다.

2

DNS 조회

브라우저는 도메인에 대응하는 서버 IP를 찾는다.

3

서버 요청

브라우저가 서버에 “페이지를 보여줘”라고 HTTP 요청을 보낸다.

4

HTML/CSS/JS 응답

서버가 화면을 그리는 데 필요한 기본 파일들을 내려준다.

5

브라우저 렌더링

브라우저가 HTML을 읽고 CSS를 적용해 화면을 그린다.

6

사용자 상호작용

클릭, 입력, 스크롤 같은 이벤트가 발생한다.

7

API 통신 후 리렌더링

이후에는 필요한 데이터만 API로 주고받고, 화면 일부만 다시 업데이트.

처음 접속할 때

브라우저는 페이지를 구성하는 재료인 HTML, CSS, JavaScript를 먼저 받는다.
즉 웹사이트 뼈대와 동작 도구를 내려받는 단계.

이후 상호작용할 때

모든 화면을 새로 받지 않고, 필요한 데이터만 API로 주고받으며 부분적으로 갱신.
우리가 방금 본 좋아요 버튼과 API테스트가 바로 이 흐름.

            5. JavaScript
            

              왜 JavaScript를 특히 알아야
              할까?
            

            

              HTML은 구조, CSS는 스타일, JavaScript는 동작을 담당. FE에서 화면을
              실제로 움직이게 만드는 중심이 JavaScript.
            

          

          

            

              
HTML

              

                제목, 문단, 버튼 같은 화면의 구조를 만든다.
              

            

            

              
CSS

              

                색상, 간격, 배치처럼 보이는 방식을 꾸민다.
              

            

            

              
JavaScript

              

                클릭, 입력, 변경, 요청, 화면 갱신 같은
                실제 동작을 만든다.
              

            

          

          

            

              

                왜 JavaScript가 특히 중요할까?
              

              

                HTML과 CSS만으로도 화면은 보일 수 있다.
                

                하지만 사용자가 버튼을 누르고, 입력을 하고, 그 결과에 따라
                내용이 바뀌는 순간부터는
                JavaScript가 반드시 필요하다.
              

              

                즉 FE에서 사용자가 체감하는 “반응하는 화면”을 만드는 핵심이
                JavaScript.
              

            

            

              
JavaScript의 DOM

              

                브라우저는 HTML을 그냥 글자로 두지 않고, 내부적으로
                구조화해서 이해한다.
              

              

                그리고 JavaScript는 그 구조를 찾아서 내용을 바꾸고, 클래스를
                바꾸고, 이벤트를 연결한다.
                JavaScript가 화면 구조를 다루는 방식이 바로 DOM이다.
              

            

          

          

            

              

                핵심
                JavaScript → DOM
              

              
HTML  → 화면의 구조
CSS   → 화면의 스타일
JS    → 화면의 동작

그리고 JS는
브라우저가 이해한 HTML 구조를
DOM이라는 방식으로 다룬다.

            

            

              
DOM을 알아보자

              

                “결국 FE가 JavaScript가 중요한 이유는 화면에 동작을 만들기 때문.
                

                그리고 이 동작은 브라우저가 이해한 HTML 구조를 다루는 방식으로
                이루어지는데, 이 구조를 JavaScript가 다루는 방법이 바로
                DOM이다.”
              

            

          

        

      

      

        

          Summary
          

            배운 것을 요약해보면?
          

          

            

              
FE

              

                사용자가 보는 화면을 만들고, 클릭/입력 같은 상호작용을 처리하고,
                서버와 통신한 결과를 다시 화면에 반영한다.
              

            

            

              
API

              

                FE와 BE가 데이터를 주고받기 위한 약속된 창구이며,
                요청(Request)과 응답(Response) 형태로 통신이 이루어진다.
              

            

            

              
웹 개발 흐름

              

                URL 입력 → DNS 조회 → 서버 요청/응답 → HTML/CSS/JS 수신 → 렌더링
                → 이후 API 통신으로 부분 업데이트가 이어진다.
              

            

            

              
HTML / CSS / JS / DOM

              

                HTML은 구조, CSS는 스타일, JavaScript는 동작이고, DOM은
                JavaScript가 화면 구조를 다루기 위한 브라우저의 표현이다.
